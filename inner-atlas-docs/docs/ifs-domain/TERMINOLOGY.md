# Inner Atlas — IFS Terminology Rules

## Say / Don't Say

| SAY | NOT |
|---|---|
| part | emotion, aspect of yourself |
| active | triggered |
| blended with | controlled by |
| Self-energy | mindfulness, calm state |
| exile | wounded part, inner child |
| system | personality, psyche |
| unburdening | healing |
| protector | manager/firefighter (collectively) |
| curious | open, receptive |

## Critical: Backend Classifications NEVER Shown to Users

Backend classifications (Perfectionist, Abandoned Child, etc.) are used ONLY in:
- Assessment inference logic
- Database backend_classification field
Never in: any UI label, badge, map node, notification, or user-facing text

Users see ONLY the name they chose.
If not yet named: "Unnamed [type]" (e.g., "Unnamed Manager")

## Required Naming Moment Language

- "This part is not its [role/behavior/belief]. That's a role it took on."
- "This part has been working very hard for a very long time."
- "That's not a problem. That's loyalty." (for protective relationships)

## Cliffhangers — Never Close a Loop

ALWAYS end with an opening:
- "There's more here. [Mini-assessment] can reveal what's waiting."
- "What made this connection necessary? That question is worth following."

NEVER say:
- "You've found all your parts."
- "Your map is complete."
